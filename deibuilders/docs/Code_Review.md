# Code Review Reports

## A. Reviewer Report

During Sprint 4, our team evaluated another group’s code in class as part of professional peer‑review practices.

### Key Observations & Suggestions
- Improve page navigation for smoother user experience  
- Adjust wording on buttons and labels to make them easier for all users to understand  
- Ensure consistent styling and spacing across pages  
- Suggest adding descriptive comments in code for maintainability  
- Recommend using clear component naming conventions  

**Outcome:**  
We submitted our recommendations to the other group during class.

## B. Response Report

After receiving feedback on our project during class, our team reflected and implemented several changes based on the suggestions.

### Changes Made
- Enhanced color scheme for better readability and accessibility  
- Improved contrast and font size for key UI elements  
- Updated button wording to be more intuitive for users  
- Reviewed navigation flow to make the site easier to use  
- Added more descriptive comments and documentation in code for clarity  

**Conclusion:**  
The code review process strengthened our team’s approach to both user experience and code quality.
